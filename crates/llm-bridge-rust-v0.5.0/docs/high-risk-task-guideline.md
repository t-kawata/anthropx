# 高风险任务处理规范

Owner: baoyx · 版本：v1.0 · 生效日期：2026-05-21

定义：跨模块重构、公共库修改、协议兼容改动、安全逻辑修改、影响多个调用方的接口变更、高回归风险优化，默认视为高风险任务。

## 1. 必须满足

- 不得直接交给单个 `coder` 全包
- 必须经过 `reviewer`
- 建议经过 `tester`
- 优先使用 `orchestrator` 或 `swarm-coordinator`

## 2. 推荐流程

- 协调器接管任务
- 说明影响面与风险边界
- 实现与验证分离
- 保留最终审查与剩余风险说明

## 3. 反模式

- 主 Agent 自己分析、编码、验证、宣布完成
- 没有 review 就结束
- 没有回归验证就合并结论

## 4. 推荐提示词

- 使用 `swarm-coordinator` 接管 [任务]，重点关注影响面、测试覆盖、回归风险和最终审查。
